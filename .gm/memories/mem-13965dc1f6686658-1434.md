HALF-RES rendering of a transparent overlay pass (water, particles, fog volumes) for a per-pixel-FS-bound surface: render it to a half-size FBO then alpha-composite (fullscreen-tri LINEAR upsample, SRC_ALPHA/ONE_MINUS_SRC_ALPHA) over the full-res scene -- ~4x fewer FS invocations, measured ~40% frame-time cut here (17.5ms->10.6ms deck). TWO correctness traps that make it draw OVER the scene: (1) the half-res FBO has NO terrain depth, so the overlay isn't occluded by geometry in front -- FIX: blitFramebuffer the full-res scene DEPTH down into the half-res FBO's depth attachment (READ scene FBO -> DRAW half FBO, DEPTH_BUFFER_BIT, NEAREST), then draw with DEPTH_TEST on + depthMask off (test, don't write); clear COLOR ONLY (keep the blitted depth). (2) a two-sided transparent surface draws its FAR-SIDE back-faces too, which composite over everything -- FIX: re-enable back-face culling matching the scene's cull winding (here cullFace FRONT to match terrain), keep two-sided only where actually needed (e.g. underwater up-view). Gate the whole path behind a window.__flag (default on) for instant A/B + rollback, and keep the refraction/scene-copy source at FULL res sampled via uResolution=halfDims so screenUV stays correct. Coarse-mesh + octave/SSR micro-quants nibbled ~5%; the half-res STRUCTURAL pass was the 8x-bigger win -- always reach for the structural lever when a cost is diffuse per-pixel over a large fill area.
