To debug WHICH term of a multi-term shader output produces a visual defect, instrument it: add a live debug uniform (driven by a window.__global, wired in the render-uniform setter like the existing uFsCheap pattern) that makes the FS output a CHOSEN intermediate (refraction tex / reflection / fog weight / absorbed body / spec / glint) clamped to [0,1], one per debug value. Then restore the user's EXACT camera pose (read cam.pos/fwd BEFORE any reload -- a shader hot-reload or page reload RESETS the camera, losing the parked defect spot) and read the center pixel for each debug value. This ELIMINATES candidates by measurement in 1-2 dispatches instead of guess-and-tweak. Lesson applied: a 'milky white' water defect was wrongly blamed on reflection/refraction/fog for several iterations; the per-term readout showed reflection weight 0.02, fog 0, spec 0, body a fine teal -- the white was actually (a) a broad sun-glint wash (pow(reflSun,LOW_EXPONENT) covers a huge solid angle -> additive white everywhere; tighten the exponent hard) and (b) too-weak Beer absorption letting bright refracted ground pass nearly untinted. ALSO: always ground-truth that the center-pixel readback is sampling the SUBJECT, not sky/UI -- screenshot once to confirm what is at screen center before trusting a pixel metric.
